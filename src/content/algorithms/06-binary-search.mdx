<Why>
- 정렬 배열에서 for문 O(N) → 이분 탐색 **O(log N)**
- "조건 만족하는 최솟값/최댓값" = 파라메트릭 서치에도 활용
- N이 10억이어도 약 30번 비교면 끝
</Why>

<How>
`left`, `right` 범위를 잡고, 중간값과 target 비교 → 반씩 줄인다. **전제: 배열 정렬**
</How>

### 기본: 값 존재 여부

```python
def binary_search(arr, target):
    left, right = 0, len(arr) - 1   # 탐색 범위의 양 끝
    while left <= right:                # 범위가 유효한 동안
        mid = (left + right) // 2       # 중간 인덱스
        if arr[mid] == target:
            return mid                  # 찾음!
        elif arr[mid] < target:
            left = mid + 1              # target이 오른쪽에 있음
        else:
            right = mid - 1             # target이 왼쪽에 있음
    return -1                           # 못 찾음
```

def parametric(lo, hi):
    while lo < hi:                  # lo == hi가 되면 답
        mid = (lo + hi) // 2        # 중간값으로 시도
        if check(mid):              # 이 값으로 조건 만족하면
            hi = mid                # 더 작은 쪽도 가능한지 탐색
        else:
            lo = mid + 1            # 불만족 → 더 큰 쪽 탐색
    return lo                       # lo == hi == 정답
