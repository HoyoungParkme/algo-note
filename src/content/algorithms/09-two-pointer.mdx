<Why>
- 이중 for문 O(N²) → 포인터 두 개로 **O(N)**
- "합이 X인 쌍", "구간 합 ≤ K인 최대 길이" 등
- "연속 구간" 키워드 → 투포인터 or 슬라이딩 윈도우 의심
</Why>

<How>
- **양끝 시작**: 합이 X인 쌍 (정렬 필수)
- **같은 쪽 시작**: 연속 구간 합 (슬라이딩 윈도우)
</How>

### 양끝에서 — 합이 X인 쌍 (BOJ 3273)

<SolutionBlock problemId="boj-1806">

```python
arr.sort()                             # 정렬 필수!
left, right = 0, N - 1                 # 양 끝에서 시작
count = 0
while left < right:                    # 두 포인터가 만나면 종료
    s = arr[left] + arr[right]
    if s == X:                         # 합이 딱 맞으면
        count += 1
        left += 1; right -= 1          # 양쪽 다 이동
    elif s < X:                        # 합이 작으면 → 큰 쪽으로
        left += 1
    else:                              # 합이 크면 → 작은 쪽으로
        right -= 1
```

</SolutionBlock>

### 같은 쪽 — 연속 부분합 ≥ S 최소 길이 (BOJ 1806)

```python
left = 0; cur = 0                      # left와 현재 구간합
ans = float('inf')                     # 최소 길이 (초기: 무한대)
for right in range(N):                 # right를 하나씩 확장
    cur += arr[right]                  # 구간에 추가
    while cur >= S:                    # 조건 만족하면
        ans = min(ans, right-left+1)   # 길이 갱신
        cur -= arr[left]               # left를 줄여서 더 짧은 구간 시도
        left += 1
```
