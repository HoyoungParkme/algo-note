## 문제 요약

`(`와 `)`로만 이루어진 문자열이 올바른 괄호 문자열(VPS)인지 판별한다.
T개의 테스트 케이스에 대해 `YES` 또는 `NO`를 출력한다.

## 접근

스택의 가장 기본 활용. 여는 괄호는 push, 닫는 괄호는 pop으로 짝을 맞춘다.

- 여는 괄호 `(`를 만나면 스택에 push
- 닫는 괄호 `)`를 만나면:
  - 스택이 비어있으면 짝이 맞지 않음 → NO
  - 스택에서 pop (짝 매칭)
- 문자열 끝까지 처리한 후 스택이 비어있으면 YES, 남아있으면 NO

## 풀이 코드

```python
import sys
input = sys.stdin.readline

T = int(input())
for _ in range(T):
    s = input().strip()
    stack = []
    valid = True
    for c in s:
        if c == '(':
            stack.append(c)
        else:                    # ')'
            if not stack:        # 짝이 없음
                valid = False
                break
            stack.pop()
    if valid and not stack:
        print('YES')
    else:
        print('NO')
```

## 핵심 포인트

- **두 가지 실패 케이스**: ① 닫는 괄호가 왔는데 스택이 빔, ② 문자열 끝났는데 스택에 남음
- 사실 이 문제는 카운터 변수 1개로도 풀린다. 그러나 여러 종류의 괄호가 섞이면 스택이 필수
- 스택의 입문 문제 — 이후 후위표기식, NGE 등에서 같은 패턴 등장
