## 문제 요약

N개의 회의가 (시작, 끝) 시간으로 주어진다. 한 회의실에서 가능한 한 많은 회의를 배정할 때, 최대 회의 수를 구한다.

## 접근

**활동 선택 문제**의 정형 그리디. **끝나는 시간이 빠른 회의부터 선택**하는 게 최적.

증명 직관: 끝나는 시간이 빠르면 다음 회의를 시작할 여유가 가장 많이 남는다.

- 회의를 (끝, 시작) 기준으로 정렬 — 끝이 같으면 시작이 빠른 순
- 현재 종료 시각 `end`를 추적
- 다음 회의의 시작 시각이 `end` 이상이면 선택 + `end` 갱신

## 풀이 코드

```python
import sys
input = sys.stdin.readline

N = int(input())
meetings = []
for _ in range(N):
    s, e = map(int, input().split())
    meetings.append((s, e))

# 끝나는 시간 오름차순, 같으면 시작 시간 오름차순
meetings.sort(key=lambda x: (x[1], x[0]))

end = 0
count = 0
for s, e in meetings:
    if s >= end:
        count += 1
        end = e

print(count)
```

## 핵심 포인트

- **정렬 키의 두 번째 조건이 중요**: 끝이 같을 때 시작이 늦은 게 먼저 오면, 시작=끝인 회의(예: (8,8))가 같은 시각에 끝난 회의 뒤에 와야 함. 시작 오름차순으로 안전하게 정렬
- `s >= end`: 이전 회의가 끝난 **시각에 바로 다음 회의 시작 가능** (문제에서 허용)
- 시작 시간이 아니라 **끝나는 시간** 기준이 핵심 — 시작 시간 기준으로 정렬하면 반례 발생
