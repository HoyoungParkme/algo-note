## 문제 요약

V개 정점, E개 간선의 가중치 그래프가 주어진다. **최소 스패닝 트리**(모든 정점을 연결하는 간선 부분집합 중 가중치 합이 최소)의 가중치 합을 구한다.

## 접근

**크루스칼 알고리즘**: 간선을 비용 오름차순으로 정렬 후, 사이클이 생기지 않는 간선만 선택. 사이클 판별은 유니온-파인드.

- 간선 모두 정렬
- 가장 싼 간선부터 순회
- 두 끝점이 다른 집합이면 → 합치고 비용 누적
- V-1개 간선을 선택하면 종료

## 풀이 코드

```python
import sys
input = sys.stdin.readline
sys.setrecursionlimit(10**6)

V, E = map(int, input().split())
edges = []
for _ in range(E):
    a, b, c = map(int, input().split())
    edges.append((c, a, b))     # 비용을 앞에 → 정렬 편함

edges.sort()

parent = list(range(V + 1))
rank_ = [0] * (V + 1)

def find(x: int) -> int:
    if parent[x] != x:
        parent[x] = find(parent[x])
    return parent[x]

def union(a: int, b: int) -> bool:
    ra, rb = find(a), find(b)
    if ra == rb:
        return False
    if rank_[ra] < rank_[rb]:
        ra, rb = rb, ra
    parent[rb] = ra
    if rank_[ra] == rank_[rb]:
        rank_[ra] += 1
    return True

total = 0
edge_count = 0
for cost, a, b in edges:
    if union(a, b):
        total += cost
        edge_count += 1
        if edge_count == V - 1:
            break

print(total)
```

## 핵심 포인트

- **간선 튜플 순서**: `(비용, a, b)` — 비용을 첫 원소에 두면 정렬이 자연스럽게 비용 기준
- **N-1개 간선**으로 트리 완성 → early break로 약간의 시간 절약
- 크루스칼 vs 프림: 간선 적으면 크루스칼, 정점 적으면 프림. 코테는 크루스칼이 더 흔함
- 같은 패턴: BOJ 1922 네트워크 연결, 1647 도시 분할 계획
