<Why>
- 매 문제마다 같은 절차를 따르면 **실수가 줄고 속도가 빨라짐**
- 조급해서 바로 코딩하면 30분 후 다 갈아엎게 된다
- 설계 → 구현 → 검증 순서를 지키면 디버깅 시간이 1/3로 줄어든다
</Why>

<How>
### 5단계 루틴

1. **읽기 (2분)** — 문제 전체 읽기. 입출력 형식 확인. **N의 범위** 체크
2. **분류 (1분)** — 키워드로 알고리즘 후보 좁히기. "이건 BFS다", "이건 DP다" 판단
3. **설계 (3~5분)** — 예제를 손으로 따라가며 풀이 방향 잡기. 자료구조 결정. **코드 짜기 전에 주석으로 단계를 먼저 적기**
4. **구현 (10~30분)** — 설계한 순서대로 코딩. 공통 패턴(입력, adj, visit 등) 먼저 깔기
5. **검증 (3분)** — 예제 입력으로 테스트. 엣지 케이스(N=1, 최대값 등) 확인. 제출
</How>

<Template>
### 설계 단계에서 주석 먼저 쓰기 예시

```python
# 문제: N개 도시, M개 도로, 시작점에서 모든 도시 최단 거리
# → 다익스트라
#
# 1. 입력 받기 (N, M, 시작점)
# 2. adj 인접 리스트 만들기 (가중치 포함)
# 3. dist 배열 무한대로 초기화
# 4. 다익스트라 실행
# 5. dist 출력 (무한대면 INF)
#
# 이렇게 주석을 먼저 쓰고 → 각 주석 밑에 코드를 채워넣기
```
