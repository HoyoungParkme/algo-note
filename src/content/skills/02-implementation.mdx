<Why>
- 알고리즘은 안 어렵다. 근데 조건이 10개, 예외가 5개, 방향 전환이 3종류면 빠짐없이 정확하게 옮기기가 어려움
- 실수 한 번이면 틀린다 — **체크리스트와 패턴**으로 실수 줄이기
- 삼성 SW 역량테스트에서 자주 출제되는 유형
</Why>

<How>
### 구현 문제 체크리스트

- **좌표계 확인** — (행, 열) vs (x, y). 0-indexed vs 1-indexed. 문제마다 다름
- **방향 처리** — 상하좌우 dx/dy 매핑. 방향 전환(시계/반시계) 로직 미리 정의
- **범위 체크** — 격자 밖 나가는지, 배열 인덱스 초과하는지 항상 확인
- **순서 중요** — "이동 후 체크" vs "체크 후 이동"이 결과를 바꿈
- **동시 처리** — "모든 칸이 동시에 변한다" → 복사본 만들어서 처리
- **종료 조건** — while문이 무한루프 안 도는지, 언제 끝나는지 명확히
</How>

dy = [0, 1, 0, -1]

direction = (direction - 1) % 4  # 반시계 방향 90도 회전

# ─── 2D 배열 복사 (동시 처리용) ───

# 또는

### 삼성 SW 역량테스트 자주 나오는 패턴

- **격자 시뮬레이션** — 2D 맵에서 캐릭터 이동, 방향 전환, 벽 만남 처리. 뱀, 미세먼지, 낚시왕
- **BFS + 조건 분기** — 일반 BFS에 "벽 부수기 1회", "열쇠 획득" 등 상태 추가. visit을 3D로 확장
- **여러 객체 동시 이동** — 상어/물고기/로봇이 동시에. 우선순위 충돌 처리. 복사본 필수
- **완전탐색 + 시뮬레이션** — 경우의 수를 다 해보고(순열/조합), 각각 시뮬레이션 돌려서 최적 선택
