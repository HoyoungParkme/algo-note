<Why>
- 코테는 **시간 싸움** — 5문제를 풀 시간은 누구에게나 똑같이 부족
- 잘 푸는 사람의 차이는 알고리즘 실력보다 **시간 배분과 막혔을 때 대응**
- 0점보다 30점, 30점보다 70점이 낫다 — 부분 점수 전략 필수
</Why>

<How>
### 시간 배분 (3시간 5문제 기준)

1. **처음 5분: 전체 훑기** — 5문제 전부 읽고 난이도 파악. 쉬운 것부터 풀 순서 정하기
2. **1~2번: 각 20~30분** — 쉬운 문제. 여기서 시간 쓰면 안 됨. 빠르고 정확하게
3. **3번: 40~50분** — 중간 난이도. 알고리즘 선택이 핵심. 모르겠으면 일단 넘어가기
4. **4~5번: 나머지 시간** — 어려운 문제. 부분 점수라도 노리기. 완벽한 풀이보다 일단 제출
</How>

### 막혔을 때 체크리스트

- **N의 범위를 다시 본다** → 허용 복잡도가 바뀌면 접근법이 바뀜
- **작은 예시를 손으로 그려본다** → 패턴이 보임
- **반대로 생각해본다** → "최소"를 구해야 하는데 "최대부터 줄여보면?"
- **문제를 쪼갠다** → "정렬 + 이분탐색" 같이 2개 조합은 아닌지
- **10분 이상 진전 없으면 다음 문제로** → 돌아와서 풀기

### 부분 점수 전략

최적 풀이를 못 찾겠으면, **비효율적이라도 정답인 코드**를 먼저 제출하자.
O(N²)이 시간초과여도 일부 테스트 케이스는 통과해서 부분 점수를 받는 경우가 많음. **0점보다 30점이 낫다**.
